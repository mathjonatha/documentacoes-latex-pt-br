\section{Detecção de Classes \KOMAScript{}}
\seclabel{identify}

Autores de pacotes às vezes precisam detectar uma classe \KOMAScript{}. Na maioria das vezes, isto é
de pouco ou nenhum interesse para o usuário médio. Para identificar a versão de
\KOMAScript{} que está sendo usada, consulte
\DescRef{scrbase.cmd.KOMAScriptVersion} em \autoref{sec:scrbase.identify},
\DescPageRef{scrbase.cmd.KOMAScriptVersion}.

\begin{Declaration}
  \Macro{KOMAClassName}%
  \Macro{ClassName}
\end{Declaration}
\Macro{KOMAClassName} armazena o nome da classe \KOMAScript{} atualmente em
uso. Portanto, se você quiser saber se uma classe \KOMAScript{} está sendo usada, você pode testar
facilmente este comando. Em contraste, \Macro{ClassName} indica qual classe padrão
foi substituída por esta classe \KOMAScript{}.

Observe\textnote{Atenção!} neste contexto, entretanto, que a existência de
\DescRef{scrlogo.cmd.KOMAScript}\IndexCmd{KOMAScript} não pode garantir que uma
classe \KOMAScript{} está em uso. Por um lado, todos os pacotes \KOMAScript{}
definem este comando. Por outro lado, outros pacotes podem achar útil definir
o rótulo \KOMAScript{} usando este nome.%

\EndIndexGroup

\section{Entradas no Índice}
\seclabel{toc}

\KOMAScript{} fornece métodos avançados para gerar e
manipular entradas no índice. Alguns deles são baseados no
pacote \Package{tocbasic} (veja \autoref{sec:tocbasic.tocstyle} começando
em \autopageref{sec:tocbasic.tocstyle}). Outros são implementados diretamente nas
classes.

\begin{Declaration}
  \Macro{raggedchapterentry}
\end{Declaration}
Em\ChangedAt{v3.21}{\Class{scrbook}\and \Class{scrreprt}} versões anteriores de
\KOMAScript{}, era possível definir a macro \Macro{raggedchapterentry} como \Macro{raggedright} para imprimir
entradas de capítulo no índice alinhadas à esquerda em vez de justificadas. Oficialmente, este recurso não
existe mais em \KOMAScript{} desde a versão 3.21.

Na verdade, o atributo \PValue{raggedentrytext} para o estilo
de entrada do índice \PValue{tocline} do pacote \Package{tocbasic}\IndexPackage{tocbasic} é
implementado definindo a macro \Macro{ragged\PName{entry level}entry} para
\Macro{relax} ou \Macro{raggedright}. Este atributo é avaliado
verificando se a macro correspondente é definida como \Macro{raggedright}. Se
for, o texto é impresso alinhado à esquerda. Com qualquer outra definição, nenhuma
margem irregular é usada.

Como foi documentado previamente que \Macro{raggedchapterentry} não deve
ser definida como nada mais que \Macro{raggedright}, este comportamento é
compatível com o comportamento documentado de versões anteriores. Como observado
nos lançamentos anteriores, outras definições de \Macro{raggedchapterentry}\,---\,e
agora também de \Macro{raggedsectionentry} e macros similares para outros níveis
de entrada\,---\,podem levar a resultados inesperados.

Você deve selecionar a justificação desejada para todas as
entradas do índice usando o atributo \PValue{raggedentrytext} de \PValue{tocline}
em vez de tentar definir o atributo para níveis de entrada específicos.%
\EndIndexGroup


\begin{Declaration}
  \Macro{addtocentrydefault}\Parameter{nível}\Parameter{número}%
  \Parameter{título}
\end{Declaration}
O\ChangedAt{v3.08}{\Class{scrbook}\and \Class{scrreprt}\and
  \Class{scrartcl}} \KOMAScript{} não usa
\Macro{addcontentsline}\IndexCmd{addcontentsline}%
\important{\Macro{addcontentsline}} diretamente para fazer entradas no índice.
Em vez disso, ele chama \Macro{addtocentrydefault} com argumentos similares. O comando
pode ser usado tanto para entradas numeradas quanto não numeradas.
O \PName{nível} é o nível de seção, isto é, \PValue{part},
\PValue{chapter}, \PValue{section}, \PValue{subsection},
\PValue{subsubsection}, \PValue{paragraph}, ou \PValue{subparagraph}. O
número da seção formatado é dado no segundo argumento, \PName{número}. Este
argumento pode estar vazio. O texto da entrada é dado pelo argumento \PName{título}. Você deve proteger
comandos frágeis dentro deste argumento com
\Macro{protect}\IndexCmd{protect}\important{\Macro{protect}}.

Uma característica notável do argumento \PName{número} é que um argumento vazio
indica que uma entrada não numerada deve ser gerada. Por padrão,
\KOMAScript{} usa
\begin{quote}
  \Macro{addcontentsline}\PParameter{toc}\Parameter{nível}%
  \Parameter{título}
\end{quote}
para isto. Se o argumento não estiver vazio, entretanto, uma entrada com número
será criada e \PName{número} é o número de título
formatado. \KOMAScript{} usa
\begin{quote}\raggedright
  \Macro{addcontentsline}\PParameter{toc}\Parameter{nível}%
  \PParameter{\%\\
    \quad\Macro{protect}\DescRef{tocbasic.cmd.numberline}\Parameter{número}%
    \PName{título}\%\\
  }
\end{quote}
para criar esta entrada.

Autores de pacotes e autores de classes envolventes podem redefinir este comando para
alterar as entradas. Por exemplo\textnote{Exemplo}, você poderia usar
\begin{lstcode}
  \renewcommand{\addtocentrydefault}[3]{%
    \IfArgIsEmpty{#3}{%
      \IfArgIsEmpty{#2}{%
        \addcontentsline{toc}{#1}{#3}%
      }{%
        \addcontentsline{toc}{#1}{\protect\numberline{#2}#3}%
      }%
    }%
  }%
\end{lstcode}
para\IndexCmd{IfArgIsEmpty} omitir entradas com um \PName{título} vazio. Na
prática, tal alteração não é necessária porque as classes \KOMAScript{}
já usam outro método para suprimir entradas vazias. Veja a descrição dos
comandos de seção em \autoref{sec:maincls.structure}, começando em
\DescPageRef{maincls.cmd.part} para isto.%
%
\EndIndexGroup


\begin{Declaration}
  \Macro{addparttocentry}\Parameter{número}\Parameter{título}
  \Macro{addchaptertocentry}\Parameter{número}\Parameter{título}
  \Macro{addsectiontocentry}\Parameter{número}\Parameter{título}
  \Macro{addsubsectiontocentry}\Parameter{número}\Parameter{título}
  \Macro{addsubsubsectiontocentry}\Parameter{número}\Parameter{título}
  \Macro{addparagraphtocentry}\Parameter{número}\Parameter{título}
  \Macro{addsubparagraphtocentry}\Parameter{número}\Parameter{título}
\end{Declaration}%
O\ChangedAt{v3.08}{\Class{scrbook}\and \Class{scrreprt}\and
  \Class{scrartcl}} \KOMAScript{} chama o comando anteriormente descrito
\DescRef{\LabelBase.cmd.addtocentrydefault}\IndexCmd{addtocentrydefault}%
\important{\DescRef{\LabelBase.cmd.addtocentrydefault}} diretamente apenas se nenhum
comando individual para o \PName{nível} foi definido ou se aquele comando
é \Macro{relax}\IndexCmd{relax}\important{\Macro{relax}}. Por padrão, todos
estes comandos simplesmente passam seu próprio \PName{nível} e argumentos diretamente para
\DescRef{\LabelBase.cmd.addtocentrydefault}.%
%
\EndIndexGroup


\section{Configurações de Fonte}
\seclabel{fonts}

\KOMAScript{} fornece não apenas uma seleção estendida de tamanhos
de fonte básica, mas também permite que você defina, manipule e aplique
elementos com suas próprias configurações de fonte.

% Note: Instead of
% \LoadNonFree{scrbookreportarticle-experts}{5}
% here is the content of that file:
\begin{Declaration}
  \Macro{@fontsizefilebase}%
  \Macro{changefontsizes}\Parameter{tamanho de fonte}
\end{Declaration}
O prefixo \File{scrsize} para nomes de arquivo de arquivos de tamanho de fonte descrito
em \autoref{sec:maincls-experts.addInfos}\iffree{}{ em
\DescPageRef{maincls-experts.option.fontsize}} é apenas o padrão para a
macro interna \Macro{@fontsizefilebase}\IndexCmd{@fontsizefilebase}. Este
padrão é usado apenas quando a macro não está ainda definida ao carregar uma
classe \KOMAScript{} ou o pacote \Package{scrextend}. Autores de classes envolventes
podem redefinir esta macro para usar arquivos de tamanho de fonte completamente diferentes.
De forma similar\textnote{Dica!} autores de classes envolventes podem mudar ou desativar
a solução de fallback para tamanhos de fonte desconhecidos redefinindo a
macro \Macro{changefontsizes}\important{\Macro{changefontsizes}}. Esta macro
tem exatamente um argumento: o \PName{tamanho de fonte} desejado.
Entretanto\textnote{Atenção!}, a macro \Macro{changefontsizes} não é
projetada como uma instrução de usuário final.%
\EndIndexGroup


\begin{Declaration}
  \Macro{newkomafont}\OParameter{aviso}\Parameter{elemento}%
                     \Parameter{padrão}%
  \Macro{aliaskomafont}\Parameter{alias}\Parameter{elemento}
\end{Declaration}
Especialistas podem usar \Macro{newkomafont} para definir um \PName{padrão} para o estilo
de fonte de um \PName{elemento}. Posteriormente, esse padrão pode ser alterado com
os comandos \DescRef{maincls.cmd.setkomafont} e \DescRef{maincls.cmd.addtokomafont}
(veja \autoref{sec:maincls.textmarkup},
\DescPageRef{maincls.cmd.setkomafont}). Claro, apenas definir o estilo
de fonte não coloca-o em uso realmente. Você deve garantir que você inclua o
comando \DescRef{maincls.cmd.usekomafont}%
\important{\DescRef{maincls.cmd.usekomafont}}\IndexCmd{usekomafont} (veja
\DescPageRef{maincls.cmd.usekomafont}) para este \PName{elemento} em seu
código nos lugares apropriados. Chamar \Macro{newkomafont} para um elemento
existente resultará em mensagens de erro.

O argumento \PName{aviso} opcional define uma mensagem de aviso. As
classes \KOMAScript{} a emitem com \Macro{ClassWarning}, e o
pacote \Package{scrextend} com \Macro{PackageWarning}, sempre que o padrão
de estilo de fonte daquele \PName{elemento} for alterado. O pacote \Package{scrkbase}
é listado como o gerador do aviso.

O comando \Macro{aliaskomafont} define um \PName{alias} para um \PName{elemento}
anteriormente definido. \KOMAScript{} informa o usuário no arquivo \File{log}
sobre o nome real do elemento se usar um \PName{alias}.
Você\textnote{Dica!} pode usar um \PName{alias}, por exemplo, se pensar em
um nome melhor para um elemento e o nome antigo deveria permanecer utilizável para
manter compatibilidade. Isto pode também aumentar a amigabilidade do usuário criando
aliases para todos os termos que diferentes usuários podem intuitivamente escolher.
O próprio \KOMAScript{} faz uso desta possibilidade.%
\EndIndexGroup


\begin{Declaration}
  \Macro{addtokomafontrelaxlist}\Parameter{macro}
  \Macro{addtokomafontonearglist}\Parameter{macro}
  \Macro{addtokomafontgobblelist}\Parameter{macro}
\end{Declaration}
Como já mencionado em \autoref{part:forAuthors} deste manual, configurações de fonte
de elementos devem consistir apenas de comandos para selecionar o tamanho,
família, codificação, série, forma, ou cor. Mudanças de cor nem
sempre são transparentes em \LaTeX{} e portanto podem causar efeitos colaterais indesejados
se você usar \DescRef{maincls.cmd.usekomafont} em um lugar inapropriado.

Usuários tendem a colocar coisas muito diferentes, às vezes críticas, na configuração
de fonte de um elemento, como \Macro{MakeUppercase} bem no final da
configuração. Na medida do possível, o uso interno das configurações de fonte foi
implementado de forma que muitos destes elementos proibidos ainda não causem dano, e
geralmente funciona mesmo se o último comando em uma configuração de fonte espera um argumento,
por exemplo usando \Macro{textbf} em vez de \Macro{bfseries}. Mas não há
garantia para isto.

Internamente, \KOMAScript{} às vezes deve limitar mudanças de fonte a configurações de fonte reais.
Isto é realizado, por exemplo, usando
\DescRef{maincls.cmd.usefontofkomafont}%
\IndexCmd{usefontofkomafont}%
\IndexCmd{usesizeofkomafont}\IndexCmd{useencodingofkomafont}%
\IndexCmd{usefamilyofkomafont}\IndexCmd{useseriesofkomafont}%
\IndexCmd{useshapeofkomafont} em vez de \DescRef{maincls.cmd.usekomafont}
(veja \autoref{sec:maincls.textmarkup},
\DescPageRef{maincls.cmd.usefontofkomafont}).

Não obstante, o comando \DescRef{maincls.cmd.usefontofkomafont} e seus
irmãos têm suas limitações. Portanto você não deve usar um comando que
sempre precisa de um argumento completamente expansível dentro da configuração
de fonte de um elemento. Mas isto é exatamente o que \Macro{MakeUppercase} precisa.
Portanto\ChangedAt{v3.17}{\Class{scrbook}\and \Class{scrreprt}\and
\Class{scrartcl}} \KOMAScript{} mantém uma lista interna de macros que
deveriam se tornar \Macro{relax} dentro de \DescRef{maincls.cmd.usefontofkomafont} e
seus irmãos. Desde \KOMAScript~3.24\ChangedAt{v3.24}{\Class{scrbook}\and
\Class{scrreprt}\and \Class{scrartcl}} apenas \Macro{normalcolor} é adicionado a
esta lista por padrão.

Observe que a \PName{macro} dada é simplesmente definida como \Macro{relax}. Então
quaisquer argumentos dentro da configuração de fonte serão executados localmente, se aplicável.
Portanto você nunca deve adicionar comandos como \Macro{setlength} à lista.
Você é responsável por todos os erros resultantes do uso de
\Macro{addtokomafontrelaxlist}. Também, não considere este comando como permissão para
adicionar todos os tipos de comandos às configurações de fonte!

Para\ChangedAt{v3.24}{\Class{scrbook}\and \Class{scrreprt}\and
\Class{scrartcl}} comandos cujo primeiro argumento deveria ser executado sem um
grupo adicional, há \Macro{addtokomatfontonearglist}. A macro especificada
é definida como \Macro{@firstofone}. Por padrão \Macro{MakeUppercase} e
\Macro{MakeLowercase} são adicionadas a esta lista.

Se\ChangedAt{v3.19}{\Class{scrbook}\and \Class{scrreprt}\and
\Class{scrartcl}}, por outro lado, uma \PName{macro} e seu primeiro argumento
deveriam ser ignorados localmente dentro de \DescRef{maincls.cmd.usefontofkomafont} e
seus irmãos, você pode usar \Macro{addtokomafontgobblelist} em vez de
\Macro{addtokomafontrelaxlist}. Um exemplo disto é o comando
\Macro{color}, que deve ser ignorado junto com o nome da cor e portanto
é um membro desta lista por padrão.

Observe que os padrões destas três listas podem mudar em versão futuras. Se você
precisa de certos comandos em uma das listas, você deveria explicitamente adicioná-los
você mesmo.%
\EndIndexGroup


\begin{Declaration}
  \Macro{IfExistskomafont}\Parameter{elemento}\Parameter{código então}%
                          \Parameter{código senão}
  \Macro{IfIsAliaskomafont}\Parameter{elemento}\Parameter{código então}%
                           \Parameter{código senão}
\end{Declaration}
Quais\ChangedAt{v3.15}{\Class{scrbook}\and \Class{scrreprt}\and
  \Class{scrartcl}\and \Class{scrlttr2}} elementos estão definidos depende da
versão de \KOMAScript{}. Então às vezes faz sentido testar antecipadamente se
um \PName{elemento} específico ainda existe. O comando \Macro{IfExistskomafont}
executa o \PName{código então} se e somente se um \PName{elemento} foi
definido usando \DescRef{\LabelBase.cmd.newkomafont} ou
\DescRef{\LabelBase.cmd.aliaskomafont} e portanto também pode ser alterado usando
\DescRef{maincls.cmd.setkomafont} ou \DescRef{maincls.cmd.addtokomafont} e
pode ser usado por um dos comandos \Macro{use\dots komafont}. Caso contrário ele
executa o \PName{código senão}.%

Em contraste, \Macro{IfIsAliaskomafont}\ChangedAt{v3.25}{\Class{scrbook}\and
\Class{scrreprt}\and \Class{scrartcl}\and \Class{scrlttr2}} executa
\PName{código então} apenas se \PName{elemento} foi definido com
\DescRef{\LabelBase.cmd.aliaskomafont} como um alias de outro elemento. Para
elementos indefinidos bem como para elementos definidos com
\DescRef{\LabelBase.cmd.newkomafont}, entretanto, ele executa o
\PName{código senão}.%

\section{Recuo de Parágrafo ou Espaço}
\seclabel{parskip}

Porque as classes \KOMAScript{} oferecem recursos estendidos para definir o
recuo de parágrafo ou espaço, mudanças diretas das extensões padrão
\Length{parskip}\IndexLength{parskip}, \Length{parindent}\IndexLength{parskip}
e \Length{parfillskip}\IndexLength{parskip} não são apenas desnecessárias mas
na maioria das vezes desaconselhadas.

\begin{Declaration}
  \Macro{setparsizes}\Parameter{recuo}\Parameter{distância}
                     \Parameter{espaço final de última linha}
\end{Declaration}
\KOMAScript{} fornece a opção de definir o recuo da primeira linha de um novo
parágrafo, a distância entre parágrafos, e o espaço em branco necessário na
linha final de cada parágrafo. Você deveria usar este comando sempre que
a opção \OptionValueRef{maincls}{parskip}{relative} deva reconhecer estas
mudanças. \KOMAScript{}\textnote{exemplo} ele próprio usa este comando, por
exemplo, na forma
\begin{lstcode}
  \setparsizes{0pt}{0pt}{0pt plus 1fil}
\end{lstcode}
para eliminar tanto o recuo de parágrafo quanto o espaçamento entre parágrafos, bem como
para permitir qualquer quantidade de espaço em branco no final da última linha do
parágrafo. Tais valores são úteis se um parágrafo consiste apenas de uma caixa que
deveria ser impressa sem espaçamento vertical e preenchendo toda a largura da coluna.
Se, por outro lado, a caixa deveria apenas preencher toda a largura mas usar
as configurações atuais de recuo e distância entre parágrafos, então
\begin{lstcode}
  \setlength{\parfillskip}{0pt plus 1fil}
\end{lstcode}
é preferível.

Começando com \KOMAScript~3.17\ChangedAt{v3.17}{\Class{scrbook}\and
  \Class{scrreprt}\and \Class{scrartcl}}, recalcular ou
reativar\IndexCmd{activateareas} a área de tipo ou as margens (veja
\autoref{cha:typearea}) também reajusta os valores de \Macro{setparsizes} se
eles não foram alterados enquanto isto. Esta é uma razão a mais não
mudar estes valores sem usar \KOMAScript{}. Definir compatibilidade com uma
versão \KOMAScript{} anterior a 3.17 (veja
\autoref{sec:maincls.compatibilityOptions},
\DescPageRef{maincls.option.version}, opção \DescRef{maincls.option.version}%
\IndexOption{version}\important{\OptionValueRef{maincls}{version}{3.17}})
desativa este recalcular.%
%
\EndIndexGroup

\section{Seções}
\seclabel{sections}

As classes \KOMAScript{} fornecem uma ampla gama de opções para modificar
níveis de seção e seus títulos correspondentes. Você pode até mesmo definir
novos níveis.

\begin{Declaration}
  \Macro{DeclareSectionCommand}\OParameter{atributos}\Parameter{nome}%
  \Macro{DeclareNewSectionCommand}\OParameter{atributos}\Parameter{nome}%
  \Macro{RedeclareSectionCommand}\OParameter{atributos}\Parameter{nome}%
  \Macro{ProvideSectionCommand}\OParameter{atributos}\Parameter{nome}
\end{Declaration}
Com\ChangedAt{v3.15}{\Class{scrbook}\and \Class{scrreprt}\and
  \Class{scrartcl}} estes comandos você pode ou definir um novo comando de seção,
\Macro{\PName{nome}}, ou modificar um comando de seção existente,
\Macro{\PName{nome}}. Para fazer isto, você usa o argumento opcional para definir vários
\PName{atributos}. O \PName{atributos} são uma lista separada por vírgula de
atribuições \PName{chave}=\PName{valor}. Em adição aos
atributos independentes de estilo mostrados em \autoref{tab:maincls-experts.declaresection.keys},
\autopageref{tab:maincls-experts.declaresection.keys}, há também
atributos que dependem do estilo. Atualmente os seguintes estilos estão
disponíveis:
\begin{description}\setkomafont{descriptionlabel}{}
\item[\PValue{chapter}:] \ChangedAt{v3.18}{\Class{scrbook}\and
    \Class{scrreprt}}O estilo para títulos de capítulo. Este estilo é usado por
  padrão para \DescRef{maincls.cmd.chapter}\IndexCmd{chapter} e indiretamente
  para \DescRef{maincls.cmd.addchap}\IndexCmd{addchap}. Você pode definir novos
  comandos de seção usando este estilo, mas então eles não automaticamente
  têm uma variante \Macro{add\dots}. Para configurar comandos de seção existentes ou novos, você também pode
  usar os atributos de
  \autoref{tab:maincls-experts.declarechapterstyle.keys},
  \autopageref{tab:maincls-experts.declarechapterstyle.keys}.
  O\textnote{Atenção!} comando \DescRef{maincls.cmd.addchap},

\Macro{DeclareNewSectionCommand} define um novo comando de seção. Se o mesmo \PName{nome}
já for usado por \TeX{} para algo mais, o comando
resultará em uma mensagem de erro e não definirá nada.

\Macro{ProvideSectionSommand} se comporta similarmente mas não emite uma mensagem de
erro.

\Macro{RedeclareSectionCommand}, por outro lado, pode apenas alterar um
comando existente para um comando de seção com os
\PName{atributos} especificados. Ele não verifica se \Macro{\PName{nome}} é já
um comando de seção. Ele apenas precisa ser um \PName{nome} já um comando \TeX{}.

\Macro{DeclareSectionCommand} não verifica se \PName{nome} é
um nome de comando \TeX{} existente. Ele apenas define um comando de seção
\Macro{\PName{nome}} com os \PName{atributos} especificados.

Cada comando de seção também tem um contador correspondente com o mesmo
\PName{nome} que é alocado com \Macro{newcounter} se necessário. A mesma
regra de nomenclatura se aplica à saída correspondente do contador
(\Macro{the\PName{nome}}), formato do contador (\Macro{\PName{nome}format}),
comando para gerar um cabeçalho de página (\Macro{\PName{nome}mark}), formato
do contador no cabeçalho de página (\Macro{\PName{nome}markformat}), elemento
de fonte (\FontElement{\PName{nome}}), e número de profundidade de seção
(\Macro{\PName{nome}numdepth}). O comando para o cabeçalho de página,
\Macro{\PName{nome}mark}, é definido por padrão para não gerar um cabeçalho
de página. A saída padrão do contador \Macro{the\PName{nome}} é um número arábico.
Se o contador é definido como dependendo de outro contador com a
\PValue{counterwithin} \PName{chave}, a saída deste contador será
prefixada por um ponto.

Em\ChangedAt[2016/03]{v3.20}{\Class{scrbook}\and \Class{scrreprt}\and
  \Class{scrartcl}} adição ao comando de seção em si, um comando para
entradas correspondentes no índice também é definido. Isto é feito
usando o pacote \hyperref[cha:tocbasic]{\Package{tocbasic}}%
\important{\hyperref[cha:tocbasic]{\Package{tocbasic}}}\IndexPackage{tocbasic}. O
atributo \PName{tocstyle} define o estilo daquelas entradas. Se
você definir um \PName{nome} vazio, ex. usando \OptionValue{tocstyle}{} ou
\OptionValue{tocstyle}{\{\}}, o comando para a entrada de índice não será
alterado. Isto é importante, por exemplo, se você usar outro pacote para modificar
o índice. Se você não definir o atributo \PValue{tocstyle}, o
estilo anterior será usado novamente durante a redefinição.

O\ChangedAt[2016/03]{v3.20}{\Class{scrbook}\and \Class{scrreprt}\and
  \Class{scrartcl}} diferentes estilos de entrada de índice também têm atributos
adicionais diferentes. Você pode defini-los diretamente se prefixá-los com \PValue{toc}.
Por exemplo, você pode definir o nível das entradas do índice, \PValue{level}, usando
\PValue{toclevel}, a indentação, \PValue{indent}, usando \PValue{tocindent},
ou a largura do número, \PValue{numwidth}, usando \PValue{tocnumwidth}. Para mais
atributos de estilo de entrada de índice veja \autoref{sec:tocbasic.tocstyle},
\autopageref{sec:tocbasic.tocstyle}.

\begin{Example}
  Por alguma razão desconhecida, você quer redefinir os
  títulos \DescRef{maincls.cmd.paragraph} para que não sejam mais
  títulos embutidos mas semelhantes aos \DescRef{maincls.cmd.subsubsection}. O
  espaço vertical acima do título deveria ser 10\Unit{pt} e nenhum espaço
  vertical adicional abaixo do título. Para fazer isto, você pode usar:
\begin{lstcode}
  \RedeclareSectionCommand[%
    beforeskip=-10pt,%
    afterskip=1sp%
  ]{paragraph}
\end{lstcode}
  O valor negativo de \PValue{beforeskip} cria um espaço vertical positivo
  antes do título e simultaneamente desabilita o recuo de parágrafo do
  texto seguinte. Embora a especificação não desejasse nenhum espaço
  vertical após o título, um valor de 1\Unit{sp} foi dado aqui. Isto é
  porque \LaTeX{} não reconhece 0\Unit{pt} como valor positivo. Então
  1\Unit{sp} é o menor valor positivo possível.

  Em geral, é melhor ter alguma tolerância para ajustar o espaçamento
  vertical, a chamada \emph{cola}:
\begin{lstcode}
  \RedeclareSectionCommand[%
    beforeskip=-10pt plus -2pt minus -1pt,%
    afterskip=1sp plus -1sp minus 1sp%
  ]{paragraph}
\end{lstcode}
  Observe que a cola também alterna o sinal algébrico antes de se tornar um espaço,
  se o valor é negativo. Esta é a razão para os valores de cola negativa no
  exemplo. Adicionalmente usamos a ocasião para minimizar o espaço vertical
  após o título usando cola também.
\end{Example}

No exemplo acima, apenas tivemos que definir as chaves \PValue{beforeskip}
e \PValue{afterskip} porque desde v3.15 \KOMAScript{} definiu
\DescRef{maincls.cmd.paragraph} internamente usando
\Macro{DeclareSectionCommand}, e portanto as outras configurações podem ser adotadas
inalteradas. A definição original de \DescRef{maincls.cmd.paragraph} em
\Class{scrartcl} lê:
\begin{lstcode}
  \DeclareSectionCommand[%
    level=4,
    indent=0pt,
    beforeskip=3.25ex plus 1ex minus .2ex,
    afterskip=-1em,
    font={},
    tocindent=7em,
    tocnumwidth=4.1em,
    counterwithin=subsubsection
  ]{paragraph}
\end{lstcode}
\Class{scrreprt} e \Class{scrbook} usam valores ligeiramente diferentes.

Algumas configurações de \DescRef{maincls.cmd.chapter} dependem da
opção \DescRef{maincls.option.headings} (veja \autoref{sec:maincls.structure},
\DescPageRef{maincls.option.headings}).
\hyperref[tab:maincls.chapter.skips]{Tabela~\ref*{tab:maincls.chapter.skips}}
mostra os valores padrão destas configurações. Uma visão geral de todas as configurações é

\autoref{sec:tocbasic.tocstyle}\important{\Package{tocbasic}},

Estes\ChangedAt{v3.15}{\Class{scrbook}\and \Class{scrreprt}\and
  \Class{scrartcl}} comandos podem definir ou alterar toda uma série de
comandos de seção de uma vez. Os nomes dos comandos de seção são dados pela
\PName{lista de nomes} separada por vírgula.

Estes comandos diferem de duas outras maneiras dos comandos anteriormente descritos
que apenas definem ou alteram um único comando de seção. Primeiro, em caso de
erro\,---\,isto é se um comando já existe com
\Macro{DeclareNewSectionCommands} ou é indefinido com
\Macro{RedeclareSectionCommands}\,---\,a definição será realizada
de qualquer forma. Uma mensagem de erro apropriada será, claro, reportada
de qualquer forma.

Segundo, há outro atributo,
\important{\PValue{increaselevel}}\OptionVName{increaselevel}{inteiro}. Este
atributo altera o significado dos atributos \PValue{level} e
\PValue{toclevel} (ver \autoref{tab:maincls-experts.declaresection.keys}) de forma que
seus valores se tornam valores iniciais para o primeiro comando de seção da
\PName{lista de nomes}. Para todos os outros comandos de seção na \PName{lista
de nomes}, os valores de \PValue{level} e \PValue{toclevel} são sucessivamente
aumentados pelo valor de \PValue{increaselevel}. Se o
atributo \PValue{increaselevel} for usado sem atribuir um valor, 1 é
assumido.%

O comando \Macro{chapterheadstartvskip} é destinado a inserir um
espaço vertical antes do título do capítulo. Similarmente,
\Macro{chapterheadendvskip} é um comando destinado a inserir um espaço
vertical após o título do capítulo. Se\ChangedAt{v3.15}{\Class{scrbook}\and
  \Class{scrreprt}} o título do capítulo tem uma linha de prefixo numérico (ver opção

Os comandos \Macro{partheadstartvskip} e \Macro{partheadendvskip} inserem
espaços verticais acima e abaixo de títulos de partes. Uma quebra de página é interpretada como
parte da distância vertical. Tal quebra de página é parte das definições padrão
de \Macro{partheadendvskip} em
\Class{scrbook}\OnlyAt{\Class{scrbook}\and \Class{scrreprt}} e
\Class{scrreprt}. O
comando \Macro{partheademptypage}\ChangedAt{v3.02}{\Class{scrbook}\and
\Class{scrreprt}} produz uma página vazia após a página do título da parte
de \Class{scrbook} e \Class{scrreprt}.

Começando com \KOMAScript~3.15\ChangedAt{v3.15}{\Class{scrbook}\and

\autoref{sec:maincls.structure},

\KOMAScript~3.17\ChangedAt{v3.17}{\Class{scrbook}\and \Class{scrreprt}\and

\begin{lstcode}
  \newcommand*{\chapterheadstartvskip}{\vspace{\@tempskipa}}
  \newcommand*{\chapterheadmidvskip}{\par\nobreak\vskip\@tempskipa}
  \newcommand*{\chapterheadendvskip}{\vskip\@tempskipa}
\end{lstcode}
Estes padrões são reativados a cada vez que você usa

\OptionValueRef{maincls}{headings}{normal}, ou
\OptionValueRef{maincls}{headings}{small}. Como efeito colateral, estas opções podem

Similarmente, ele define esta extensão para o valor de
\PValue{afterskip} antes de chamar \Macro{chapterheadendvskip}, e para
\PValue{innerskip} antes de chamar \Macro{chapterheadmidvskip}.

Os valores padrão das distâncias de \DescRef{maincls.cmd.part} não
dependem da opção \DescRef{maincls.option.headings}. Então os comandos correspondentes
não serão redefinidos por esta opção.

    \par\nobreak
    \vskip\@tempskipa

  \newcommand*{\partheadendvskip}{%

\begin{lstcode}

\begin{Declaration}
  \Macro{partlineswithprefixformat}%
    \Parameter{nível}\Parameter{número}\Parameter{texto}%
\end{Declaration}
Este\ChangedAt{v3.25}{\Class{scrbook}\and \Class{scrreprt}\and
  \Class{scrartcl}} comando é usado por títulos com o estilo \PValue{part} para
emitir o número do título e texto do título. Os argumentos \PName{número} e
\PName{texto} já estão formatados, incluindo as
seleções de fonte. Finalmente, este comando controla o arranjo das duas partes
do título. Para títulos não numerados, \PName{número} é um
argumento completamente vazio, então ele não contém nenhum comando de formatação.

A definição padrão é bastante simples:
\begin{lstcode}
  \newcommand{\partlineswithprefixformat}[3]{%
    #2#3%
  }
\end{lstcode}

\begin{Example}
  Você quer ter títulos de partes em uma caixa azul clara com moldura azul. A
  caixa deveria ocupar apenas cerca de três quartos da largura completa da
  área de texto. Então você tenta:
\begin{lstcode}
  \documentclass{scrbook}
  \usepackage{xcolor}
  \renewcommand*{\partlineswithprefixformat}[3]{%
    \fcolorbox{blue}{blue!25}{%
      \parbox{.75\linewidth}{#2#3}%
    }%
  }
  \begin{document}
  \part{Framed Part}
  \end{document}
\end{lstcode}
  Mas surpreendentemente o título não está mais centralizado\,---\,nem a caixa
  em si nem o texto dentro da caixa.

  A razão para a falta de centralização da caixa é que o final do
  parágrafo está oculto no terceiro argumento do comando. Então ele ainda
  termina o parágrafo do texto dentro da caixa mas não do parágrafo da
  \Macro{parbox} em si. Para resolver isto você adiciona um \Macro{par} ao final da
  definição.

  A razão para a falta de centralização dentro da caixa é que o alinhamento de
  \DescRef{maincls.cmd.raggedpart} é válido fora da caixa mas não
  automaticamente dentro de uma \Macro{parbox}. Para resolver isto você adiciona
  \Macro{raggedpart} dentro da caixa.

  Com
\begin{lstcode}
  \documentclass{scrbook}
  \usepackage{xcolor}
  \renewcommand*{\partlineswithprefixformat}[3]{%
    \fcolorbox{blue}{blue!25}{%
      \parbox{.75\linewidth}{\raggedpart #2#3}%
    }%
  }
  \par
  \begin{document}
  \part{Framed Part}
  \end{document}
\end{lstcode}
  você obtém o resultado esperado.
\end{Example}

Como o exemplo mostra, usuários que redefinem este comando devem observar
vários efeitos colaterais. Além de preservar o alinhamento de texto, eles também
devem impedir quebras de página dentro dos títulos, por exemplo se inserirem parágrafos
extras ou espaço. O exemplo acima não tem este problema. Não apenas
a caixa impede quebras de página de qualquer forma, mas \KOMAScript{} mesmo
altera \Macro{interlinepenalty} como parte de \PName{texto} para impedir quebras
de página lá. Ele também termina \PName{texto} com uma quebra de parágrafo interna
usando \Macro{@@par}.

A definição padrão de \Macro{partlineswithprefixformat} não usa o
primeiro argumento, \PName{nível}, nem é necessário no exemplo acima. É de
interesse apenas se você quer definir vários comandos com o estilo \PValue{part}
e precisa distinguir os diferentes níveis. Os comandos predefinidos
\DescRef{maincls.cmd.part}, \DescRef{maincls.cmd.part*},
\DescRef{maincls.cmd.addpart}, e \DescRef{maincls.cmd.addpart*} todos compartilham
o mesmo \PName{nível} \PName{part}.%

\begin{Declaration}

\end{Declaration}
Estes\ChangedAt{v3.19}{\Class{scrbook}\and \Class{scrreprt}} comandos são usados

texto do título, dependendo da

\textnote{\DescRef{maincls.option.chapterprefix}} (veja

\begin{Example}
  Você quer ter títulos de capítulo com fundo amarelo. Para os títulos
  sem uma linha de prefixo, você usa a seguinte definição no preâmbulo
  do documento:
\begin{lstcode}
  \makeatletter
  \renewcommand{\chapterlinesformat}[3]{%
    \colorbox{yellow}{%
      \parbox{\dimexpr\linewidth-2\fboxrule-2\fboxsep}{%
        \@hangfrom{#2}#3%
      }%
    }%
  }
  \makeatother
\end{lstcode}
  Para títulos de capítulo com linha de prefixo, você usa:
\begin{lstcode}
  \renewcommand{\chapterlineswithprefixformat}[3]{%
    \colorbox{yellow}{%
      \parbox{\dimexpr\linewidth-2\fboxrule-2\fboxsep}{%
        #2#3%
      }%
    }%
  }
\end{lstcode}
  Infelizmente, você descobre que estas redefinições resultam em texto
  justificado para os títulos. A razão é o comando \Macro{parbox}. Para mudar
  isto, você pode usar o comando \DescRef{maincls.cmd.raggedchapter} (veja
  \autoref{sec:maincls.structure}, \DescPageRef{maincls.cmd.raggedchapter})
  dentro do argumento de \Macro{parbox}. Caso contrário \Macro{raggedchapter} seria
  usado apenas antes de \Macro{chapterlineswithprefixformat} e
  \Macro{chapterlinesformat}:
\begin{lstcode}
  \makeatletter
  \renewcommand{\chapterlinesformat}[3]{%
    \colorbox{yellow}{%
      \parbox{\dimexpr\linewidth-2\fboxrule-2\fboxsep}{%
        \raggedchapter
        \@hangfrom{#2}#3%
      }%
    }%
  }
  \makeatother
  \renewcommand{\chapterlineswithprefixformat}[3]{%
    \colorbox{yellow}{%
      \parbox{\dimexpr\linewidth-2\fboxrule-2\fboxsep}{%
        \raggedchapter
        #2#3%
      }%
    }%
  }
\end{lstcode}
  Lembre-se de usar \Macro{makeatletter} e \Macro{makeatother} apenas no
  preâmbulo do documento. Não o use dentro de sua própria classe envolvente ou pacote.
  Eles são necessários apenas por causa de \Macro{@hangfrom} na definição de
  \Macro{chapterlinesformat}.
\end{Example}

Como o exemplo mostra, usuários que redefinem este comando devem observar
vários efeitos colaterais. Além de preservar o alinhamento de texto, eles também
devem impedir quebras de página dentro dos títulos, por exemplo se inserirem parágrafos
extras ou espaço. O exemplo acima não tem este problema. Não apenas
a caixa impede quebras de página de qualquer forma, mas \KOMAScript{} mesmo
altera \Macro{interlinepenalty} como parte de \PName{texto} para impedir quebras
de página lá. Ele também termina \PName{texto} com uma quebra de parágrafo interna
usando \Macro{@@par}.

Incidentalmente, o comando \DescRef{maincls.cmd.raggedchapter} não é parte de
\PName{texto}, como caso contrário usar \Macro{MakeUppercase} dentro de versões redefinidas
destes dois comandos seria muito mais difícil. Observe, entretanto,
que regras tipográficas requerem ajustes individuais do espaçamento de letras em
texto capitalizado. Entretanto o comando \LaTeX{} \Macro{MakeUppercase} não
faz isto.

As definições padrão não usam o primeiro argumento, \PName{nível}, nem é
necessário no exemplo acima. É de interesse apenas se você quer definir
vários comandos com o estilo \PValue{chapter} e precisa distinguir os
diferentes níveis. Os comandos predefinidos \DescRef{maincls.cmd.chapter},
\DescRef{maincls.cmd.chapter*}, \DescRef{maincls.cmd.addchap}, e
\DescRef{maincls.cmd.addchap*} todos compartilham o mesmo \PName{nível}
\PName{chapter}.%
\EndIndexGroup


\begin{Declaration}
  \Macro{sectionlinesformat}%
    \Parameter{nível}\Parameter{recuo}\Parameter{número}\Parameter{texto}%
  \Macro{sectioncatchphraseformat}%
    \Parameter{nível}\Parameter{recuo}\Parameter{número}\Parameter{texto}%
\end{Declaration}
Estes\ChangedAt{v3.19}{\Class{scrbook}\and \Class{scrreprt}\and
\Class{scrartcl}} comandos são usados por títulos com o estilo \PValue{section}
dependendo se o título é embutido ou em sua própria linha.
Títulos independentes usam \Macro{sectionlinesformat} para sua saída, enquanto
títulos embutidos usam \Macro{sectioncatchphraseformat}.

Em ambos os casos \PName{recuo} é o valor da indentação horizontal do
título relativa à área de texto. Se o valor é negativo, o título pode
protrair na margem esquerda.

Os argumentos \PName{número} e \PName{texto} já estão formatados, incluindo
as configurações de fonte. Finalmente, estes comandos controlam o arranjo das
duas partes do título. Para títulos não numerados, o argumento \PName{número}
é completamente vazio, então ele não contém nenhum comando de formatação.

As definições padrão são:
\begin{lstcode}
\newcommand{\sectionlinesformat}[4]{%
  \@hangfrom{\hskip #2#3}{#4}%
}
\newcommand{\sectioncatchphraseformat}[4]{%
  \hskip #2#3#4%
}
\end{lstcode}

Se você redefinir um destes comandos, você é responsável por impedir quebras de
página dentro do título. \KOMAScript{} mesmo apenas altera
\Macro{interlinepenalty} para impedir quebras de página.

\begin{Example}
  Como no exemplo anterior com títulos de capítulo, os títulos independentes
  do nível \PValue{section} deveriam agora ser impressos com uma cor de
  fundo. Títulos de níveis mais baixos não deveriam ser alterados:
\begin{lstcode}
  \makeatletter
  \renewcommand{\sectionlinesformat}[4]{%
    \@tempswafalse
    \Ifstr{#1}{section}{%
      \hspace*{#2}%
      \colorbox{yellow}{%
        \parbox{\dimexpr\linewidth-2\fboxrule-2\fboxsep-#2}{%
          \raggedsection
          \@hangfrom{#3}{#4}%
        }%
      }%
    }{%
      \@hangfrom{\hskip #2#3}{#4}%
    }%
  }
  \makeatother
\end{lstcode}
  Com este código, área da indentação não é colorida se o título é
  indentado. Se, entretanto, o título é colocado na margem esquerda com
  indentação negativa, esta área da margem também é destacada. Você pode
  mover o comando \Macro{hspace*} para dentro de \Macro{colorbox} para alterar este
  comportamento.

  Novamente, lembre-se de usar \Macro{makeatletter} e \Macro{makeatother} apenas no
  preâmbulo do documento. Você deveria omiti-los dentro de sua própria classe envolvente ou
  pacote. Eles são necessários apenas por causa de \Macro{@hangfrom} na definição
  de \Macro{sectionlinesformat}.
\end{Example}

O primeiro argumento, \PName{nível}, não é usado pela definição padrão. O
exemplo mostra como usá-lo para distinguir diferentes níveis de título do
mesmo estilo \PValue{section}.%

\DescRef{\LabelBase.cmd.DeclareSectionCommand},
\DescRef{\LabelBase.cmd.DeclareNewSectionCommand},
\DescRef{\LabelBase.cmd.ProvideSectionCommand} e
\DescRef{\LabelBase.cmd.RedeclareSectionCommand} fornecem vários ganchos nos
comandos de seção que podem ser manipulados usando
\DescRef{scrbase.cmd.AddtoDoHook}. Para informação sobre a funcionalidade de
estes \emph{do-hooks} veja \autoref{sec:scrbase.hooks},
\autopageref{sec:scrbase.hooks}. O último elemento do especificador é o
\PName{nome} do comando de seção, como é para os comandos de declaração
mencionados acima.

É importante\important{Atenção!} evitar adicionar código a estes ganchos que
afetaria a quebra de página ou a posição do título. Portanto apenas
usuários avançados deveriam usar estes ganchos. Em caso de dúvida, consulte o
código fonte da classe para informação sobre exatamente quando um gancho é executado. Estes ganchos
servem como um fallback antes de você realmente precisaria recorrer a redefinir
um comando de seção sem usar recursos \KOMAScript{}.

O\textnote{\DoHHook{\dits/preinit/\dots}} gancho
\DoHook{heading/preinit/\PName{nome}} é executado imediatamente antes do
comando de seção ser inicializado. Neste ponto, nenhuma configuração foi
inicializada. Nem mesmo o parágrafo que precede o título está
necessariamente terminado ainda.

O\textnote{\DoHook{\dots/postinit/\dots}} gancho
\DoHook{heading/postinit/\PName{nome}} é executado um pouco mais tarde. Neste
ponto, algumas configurações foram inicializadas e o parágrafo anterior já
foi encerrado.

Apenas\textnote{\DoHook{\dots/branch/\dots}} um dos ganchos
\DoHook{heading/branch/nostar/\PName{nome}} ou
\DoHook{heading/branch/star/\PName{nome}} é executado, após determinar
se a variante com ou sem asterisco do comando de seção é usada. Neste
ponto, o espaço vertical acima do título já foi inserido.

O\textnote{\DoHook{\dots/begingroup/\dots}} gancho
\DoHook{heading/begingroup/\PName{nome}} é executado no início do
grupo usado para processar o título. Esta é também a última oportunidade para
intervir antes do título ser impresso.

Similarmente\textnote{\DoHook{\dots/begingroup/\dots}}, o gancho
\DoHook{heading/endgroup/\PName{nome}} será executado no final do
grupo usado para processar o título. Atualmente este é o último gancho dentro
de um comando de seção.

Observe\textnote{\DescRef{maincls.cmd.minisec}} que
\DescRef{maincls.cmd.minisec} não é um comando de seção real e portanto
estes ganchos não estão disponíveis para ele.%

substituir alguns comandos do kernel \LaTeX{}:
\begin{itemize}
\item Em vez de
  \Macro{@startsection}\IndexCmd{@startsection}\important{\Macro{@startsection}}
  \KOMAScript{} usa \Macro{scr@startsection}. Entretanto, a definição de
  \Macro{@startsection} é verificada. Se a implementação difere da
  esperada, um aviso é emitido, vários recursos \KOMAScript{}
  são desativados, e \Macro{scr@startsection} usa uma cópia de
  \Macro{@startsection} enquanto \Macro{@startsection} é redefinido para usar
  \Macro{scr@startsection}.
\item Em vez de \Macro{@dblarg}\IndexCmd{@dblarg}\important{\Macro{@dblarg}}
  \KOMAScript{} usa um comando interno para definir comandos de seção.
\item Em vez de \Macro{secdef}\IndexCmd{secdef}\important{\Macro{secdef}},
  \KOMAScript{} usa \Macro{SecDef} para alterar \Macro{@dblarg} como
  mencionado acima. Se a implementação de \Macro{secdef} difere da
  esperada, um aviso é emitido.
\item \Macro{@sect}\IndexCmd{@sect}\important{\Macro{@sect}} é redefinido para
  implementar várias extensões de \KOMAScript{}.
\item \Macro{@ssect}\IndexCmd{@ssect}\important{\Macro{@ssect}} é redefinido
  para implementar várias extensões de \KOMAScript{}.
\item \Macro{@xsect}\IndexCmd{@xsect}\important{\Macro{@xsect}} é redefinido
  para implementar várias extensões de \KOMAScript{}.
\end{itemize}
Em versões futuras de \KOMAScript{}, está sendo considerado evitar
alterar os comandos do kernel \LaTeX{} mencionados acima e substituí-los
completamente com comandos internos personalizados. Carregar pacotes de seção
verdadeiramente incompatíveis desativaria automaticamente extensões \KOMAScript{}
e transferiria a responsabilidade pelos comandos de seção inteiramente
para estes pacotes. Por outro lado, esforço extra seria necessário
para manter compatibilidade com outros pacotes.

Autores de pacotes podem usar estes comandos como usariam os comandos correspondentes
do kernel \LaTeX{} e portanto ganhar acesso aos recursos adicionais
de \KOMAScript{}. Entretanto, estes comandos não deveriam ser redefinidos, já que podem ser
alterados a qualquer momento, e então esta redefinição poderia comprometer
a funcionalidade de \KOMAScript{}. O significado dos parâmetros para estes
comandos pode ser encontrado no manual do kernel \LaTeX{} \cite{latex:source2e}.
Como\ChangedAt[2019/01]{v3.27}{\Class{scrbook}\and \Class{scrreprt}\and
  \Class{scrartcl}} uma alternativa para redefinir tais comandos, \KOMAScript{}
oferece os ganchos descritos previamente.%

\begin{Declaration}
  \Macro{appendixmore}
\end{Declaration}%
As classes \KOMAScript{} têm um recurso incomum dentro do
comando \DescRef{maincls.cmd.appendix}. Se o comando \Macro{appendixmore}
estiver definido, \DescRef{maincls.cmd.appendix} também o executará. Internamente,
as classes \KOMAScript{} \Class{scrbook}\OnlyAt{\Class{scrbook}\and
  \Class{scrreprt}} e \Class{scrreprt} tiram vantagem deste comportamento para
implementar a opção de apresentação \DescRef{maincls.option.appendixprefix} (veja
\autoref{sec:maincls.structure},
\DescPageRef{maincls.option.appendixprefix}). Você\textnote{Atenção!} deveria
observar isto no caso de você decidir definir ou redefinir a
macro \Macro{appendixmore}. Se esta opção foi usada, você receberá uma
mensagem de erro ao usar \verb|\newcommand{\appendixmore}{|\dots\verb|}|. Este
comportamento se destina a impedir que você sobrescreva as opções sem
perceber.

\begin{Example}
  Você não quer que os capítulos na parte principal das classes \Class{scrbook}
  ou \Class{scrreprt} sejam introduzidos por uma linha de prefixo (veja a
  opção de apresentação \DescRef{maincls.option.chapterprefix} em
  \autoref{sec:maincls.structure},
  \DescPageRef{maincls.option.chapterprefix}). Para consistência, você também não
  quer tal linha no apêndice. Em vez disso, você gostaria de ver
  a palavra ``Apêndice'' no idioma de sua escolha escrita na frente da
  letra do capítulo e, simultaneamente, nos cabeçalhos de página. Em vez de usar
  a opção de apresentação \DescRef{maincls.option.appendixprefix}, você define o
  seguinte no preâmbulo do documento:
%
\begin{lstcode}
  \newcommand*{\appendixmore}{%
    \renewcommand*{\chapterformat}{%
      \appendixname~\thechapter\autodot\enskip}
    \renewcommand*{\chaptermarkformat}{%
      \appendixname~\thechapter\autodot\enskip}
  }
\end{lstcode}
%
  No caso de você depois mudar de ideia e decidir usar a opção
  \DescRef{maincls.option.appendixprefix} em um estágio posterior, você receberá uma
  mensagem de erro por causa do comando \Macro{appendixmore} já definido.
  Este comportamento impede a definição feita acima de mudar invisível
  as configurações pretendidas com a opção.

  Também é possível obter um comportamento similar para o apêndice da classe
  \Class{scrartcl}. Por exemplo, você pode escrever no preâmbulo de seu
  documento:
\begin{lstcode}[moretexcs={ifthenelse,equal}]
  \newcommand*{\appendixmore}{%
    \renewcommand*{\sectionformat}{%
      \appendixname~\thesection\autodot\enskip}%
    \renewcommand*{\sectionmarkformat}{%
      \appendixname~\thesection\autodot\enskip}}
\end{lstcode}

  Os comandos redefinidos são explicados em mais detalhes em
  \autoref{sec:maincls.structure},
  \DescPageRef{maincls.cmd.chapterformat} e
  \DescPageRef{maincls.cmd.chaptermarkformat}.
\end{Example}

\section{Bibliografia}
\seclabel{bibliography}

A informação nesta seção é menos importante se você usar pacotes como
\Package{biblatex}. Neste caso, os recursos extensivos de tais pacotes
sobrepujarão as extensões das classes \KOMAScript{} descritas aqui.

\begin{Declaration}
  \Macro{newbibstyle}\OParameter{estilo pai}\Parameter{nome}%^^A
                     \Parameter{comandos}%
  \Macro{newblock}%
  \Macro{@openbib@code}%
  \Macro{bib@beginhook}%
  \Macro{bib@endhook}
\end{Declaration}
As classes padrão\textnote{classes padrão} já fornecem o
comando \Macro{newblock} para estruturar entradas de bibliografia. O propósito exato
deste comando depende das opções da classe. Usando a opção
\Option{openbib}\important{\Option{openbib}} redefine os comandos
\Macro{@openbib@code} e \Macro{newblock} em si ao final da classe
padrão. As classes padrão executam o comando \Macro{@openbib@code} ao
iniciar a lista de bibliografia, ou mais precisamente, ao definir os
parâmetros para a lista. Você pode assumir que muitos pacotes que redefinema
bibliografia executarão este comando.

As classes \KOMAScript{}\textnote{\KOMAScript{}} fazem algo similar. Entretanto,
elas não redefinem \Macro{@openbib@code} ao final da classe. Em vez disso,
\Macro{newbibstyle} define o estilo de bibliografia \PValue{openstyle}. O
\PName{comandos} dados na implementação contêm a apropriada
redefinição de \Macro{@openbib@code} e \Macro{newblock}. Se você selecionar este
estilo de bibliografia usando a opção
\OptionValueRef{maincls}{bibliography}{openstyle}%
\IndexOption{bibliography~=\PValue{openstyle}}%
\important{\OptionValueRef{maincls}{bibliography}{openstyle}}, o
\PName{comandos} será executado imediatamente. Isto redefinirá
\Macro{@openbib@code} e \Macro{newblock}.

Em adição a \Macro{@openbib@code} e \Macro{newblock},
\Macro{bib@beginhook} e \Macro{bib@endhook} também podem ser redefinidas pelo
\PName{comandos} do estilo. O comando \Macro{bib@beginhook} é executado
imediatamente após o título e preâmbulo da bibliografia mas antes do
início da lista de entradas bibliográficas. O comando \Macro{bib@endhook}
será executado imediatamente após esta lista, ao final da
bibliografia. Se \DescRef{maincls.cmd.BreakBibliography} (veja
\autoref{sec:maincls.bibliography},
\DescPageRef{maincls.cmd.BreakBibliography}) for usado para interromper a
bibliografia, estes comandos também serão executados no início e final
de cada parte da bibliografia, imediatamente antes e depois
\DescRef{maincls.cmd.BreakBibliography}.

Os comandos \Macro{newblock}, \Macro{@openbib@code}, \Macro{bib@beginhook}, e
\Macro{bib@endhook} são inicialmente definidos como vazios ao usar um
novo estilo de bibliografia. Depois disto, o \PName{comandos} do \PName{estilo pai}
que foi opcionalmente especificado ao definir o estilo será executado,
seguido pelos \PName{comandos} para o estilo de bibliografia em si. Como
resultado, estes comandos devem ser definidos, se necessário, com
\Macro{renewcommand}\IndexCmd{renewcommand}\important{\Macro{renewcommand}},
não \Macro{newcommand}, dentro do argumento \PName{instruções}.

Se você usar os comandos
\DescRef{maincls.cmd.AtEndBibliography}\IndexCmd{AtEndBibliography} e
\DescRef{maincls.cmd.AfterBibliographyPreamble}%
\IndexCmd{AfterBibliographyPreamble} para declarar \PName{comandos} adicionais
a serem executados após o preâmbulo ou ao final da
bibliografia, o \PName{comandos} especificados com
\DescRef{maincls.cmd.AfterBibliographyPreamble} será executado apenas uma vez, no
início da bibliografia após o \Macro{bib@beginhook}
\PName{comandos}, e o \PName{comandos} de
\DescRef{maincls.cmd.AtEndBibliography} será executado apenas uma vez ao final
da bibliografia, antes de \Macro{bib@endhook}.

Para\textnote{Exemplo!} exemplo, o
pacote \Package{multicol}\IndexPackage{multicol}\important{\Package{multicol}}
(veja \cite{package:multicol}) poderia ser usado para definir um
estilo de bibliografia para imprimir a bibliografia com duas colunas:
% Umbruchkorrektur: listings korrigiert
\begin{lstcode}
  \newbibstyle{twocolumstyle}{%
    \renewcommand*{\bib@beginhook}{\begin{multicols}{2}}%
    \renewcommand*{\bib@endhook}{\end{multicols}}}%
\end{lstcode}
Se você também quiser definir uma variação aberta deste estilo, você pode usar
as possibilidades de herança aqui e especificar um \PName{estilo pai}:
% Umbruchkorrektur: listings korrigiert
\begin{lstcode}
  \newbibstyle{twocolumopenstyle}[openstyle]{%
    \renewcommand*{\bib@beginhook}{\begin{multicols}{2}}%
    \renewcommand*{\bib@endhook}{\end{multicols}}}%
\end{lstcode}
Você pode então selecionar estes estilos recém-definidos com a
opção \DescRef{maincls.option.bibliography} como usual.

Como\textnote{Atenção!} \DescRef{maincls.cmd.BreakBibliography}, estes
comandos perdem todos ou parte de seu efeito quando \Environment{thebibliography}
é redefinido, por exemplo ao usar
\Package{biblatex}\IndexPackage{biblatex}\important{\Package{biblatex}}.%
